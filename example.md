请基于一个 monorepo，从 0 到 1 生成一个“AI 电子伴侣” technical example 的第一版核心骨架代码。这个项目是技术演示样例，不是正式商用产品，所有实现优先围绕技术链路验证、架构表达、端到端主链路跑通，不要把精力放在商业化功能、复杂运营系统、精美视觉包装上。

## 一、总体目标

要做的是一个包含：

- 前台用户端（Next.js）
- 后台管理端（Next.js）
- 服务端 API（Hono + Cloudflare Workers）

的完整技术样例项目。

这个 example 的核心不是“泛 AI 助手”，而是一个 **恋爱闲聊训练 + 情绪理解 + 陪伴成长** 的 AI 电子伴侣系统。

项目需要同时覆盖两条主线，但第一版的**硬主链路**优先打通训练链路：

1. **陪伴聊天主线**
   - 用户和 AI 长期对话
   - AI 能识别情绪、召回长期记忆、维持陪伴感
   - 陪伴链路会适度承接训练结果，但不是强承接

2. **训练主线（第一版最优先打通）**
   - 用户进入恋爱闲聊训练
   - AI 扮演训练对象
   - 进行多轮练习
   - 训练结束后给评分、点评、改写建议
   - 结果写入长期记忆和能力画像
   - 成长记录页可查看训练轨迹和能力变化

外层用户感知上，这是**同一个 AI 身份**；系统内部要区分“陪伴子角色”和“训练子角色”两条内部机制。

## 二、严格遵守的实现边界

### 1. 这是 technical example，不是正式产品
所以请把重点放在这些技术能力上：

- 链路观测
- Cloudflare 基础设施映射

- 过深的权限运营体系

- 真正复杂的外部消息推送系统

### 2. 工具 / MCP
第一版**不接真实工具调用和 MCP**，但代码结构里要**预留扩展位**，让后续可继续接入。

### 3. 用户交互模态
第一版只做：
- **纯文本聊天**
- **纯文本训练**

不做语音，不做图片。

### 4. 流式协议
第一版前台聊天与训练回复，先做：
- **纯文本流式回复**

不要优先做 object stream 或复杂 message parts 渲染。

### 5. 安全与合规边界
第一版只做：
- **基础安全规则 + 基础审核**
- 不建设复杂审核平台

必须明确拦截：
- 违法内容
- 极端暴力或仇恨内容
- 诱导失控或关系操控内容
- 涉及未成年人的不当暧昧或亲密情境

输出边界：
- 不提供医疗建议
- 不提供心理诊断
- 不提供法律建议
- 定位在陪伴支持、训练反馈与沟通建议

数据与展示边界：
- 用户自定义人设与训练内容先经过基础审核，再参与生成
- 后台默认只展示摘要级观测信息，不直接展示原始对话全文
- 架构上保留后续扩展位，但第一版不做重审核体系

## 六、训练系统设计要求

### 1. 训练对象人设机制
第一版训练对象需要支持：

- 系统内置角色模板
- 用户前台填写的“自定义人设补充”

规则如下：

- 用户可在前台输入 **300 字以内** 的自定义人设文本
- 用户可保存**多个**常用人设预设
- 自定义人设**只影响训练模式**
- 训练时采用：
  - **系统模板为主**
  - **用户人设为补充**
- 进入训练时采用：
  - **系统推荐 + 用户可手动切换**

请设计相应的数据结构、前端交互和服务端校验逻辑。

### 2. 人设安全处理
对用户自定义人设输入，第一版做：
- **基础审核**
- 拦截明显违规、极端、诱导失控的人设描述
- 合法内容再参与角色生成

### 3. 训练周期
训练体系采用：
- **三阶段闯关制**
  - 入门
  - 进阶
  - 提升

每个阶段同时体现：
- 技巧目标
- 场景范围

### 4. 预置训练场景
第一版预置场景库规模：
- 约 **9 个场景**
- 分配方式：
  - 入门 4
  - 进阶 3
  - 提升 2

场景生成方式：
- **预置场景库 + 动态改写**
- 先有固定场景模板
- 再结合用户能力画像、阶段、角色模板进行细节调整

### 5. 单轮训练结束条件
采用：
- **双条件结束**
  - 最大轮次上限
  - 目标提前达成可提前结束

最大轮次：
- **10 轮以内**

## 七、训练评分体系

### 1. 固定基础维度
所有阶段都必须有这 5 个固定评分维度：

- 开场自然度
- 接话能力
- 话题延展
- 情绪感知
- 边界感

### 2. 阶段专项维度
每个阶段额外补 1 个专项维度：

- 入门：回应稳定性
- 进阶：共情深度
- 提升：暧昧表达分寸

### 3. 晋级规则
阶段通关采用：
- **总分 + 关键项达标**
- 不能只看综合分
- 关键维度如果明显偏低，不能晋级

### 4. 评分结果生成方式

- **同步返回基础评分与简评**
- **异步补充详细点评、改写建议、记忆写回结果**

前台呈现方式：
- 当前训练结果面板自动增量刷新
- 同时在通知页记录一次“补充分析完成”的站内通知
