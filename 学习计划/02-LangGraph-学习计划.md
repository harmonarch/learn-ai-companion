# LangGraph 学习计划

## 这个领域解决什么问题

LangGraph 负责显式状态管理、节点编排、条件路由、循环控制、中断恢复和多 Agent 协作，适合构建多步骤、有状态、可恢复的 Agent 工作流。

## 学习目标

- 理解 Graph 思维与 Chain 思维的差别
- 能设计 StateGraph 的 state、node、edge
- 能处理条件路由、循环和停止条件
- 能理解 checkpointer、interrupt、store 的用途
- 能完成 AI 伴侣管线的图式编排

## 核心易混点

- graph state、节点内部临时变量、长期记忆不是一回事。
- node 职责和普通辅助函数职责要分清。
- 路由条件稳定性比“能跑起来”更重要。
- checkpointer 和应用层持久化的边界要分开。

## 必做失败演练

- 故意不给循环设计停止条件，观察图为什么会死循环。
- 故意把不该进 state 的字段塞进去，检查后续节点为什么变难维护。
- 故意让 router 依赖模糊文本，观察分支为什么不稳定。

## 改造练习

- 把线性 tool agent 改造成 graph 版本，并重新划分 state、node、edge。

## 系统连接

- 上游依赖 LangChain、AI SDK 或其他原子能力层提供节点能力。
- 下游会接长期记忆、人工介入、观测和生产恢复机制。
- 在 AI 伴侣里常落在多步工作流层和长期状态控制层。

## 常见误区

1. 把所有数据都塞进 state，导致图越来越难维护
2. 节点职责过大，一个 node 里堆太多步骤
3. 没有设计停止条件，图容易进入死循环
4. 路由判断依赖模糊文本，行为不稳定
