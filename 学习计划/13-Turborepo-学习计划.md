# Turborepo 学习计划

## 这个领域解决什么问题

Turborepo 负责 Monorepo 中的任务编排、构建缓存、过滤执行和 CI 效率优化，适合管理多应用、多包协作下的开发与构建流程。

## 学习目标

- 理解任务图和缓存机制的基本思路
- 能配置最小 `turbo.json`
- 能说清 build、lint、test、dev 之间的依赖关系
- 能利用 filter 和缓存提升开发效率
- 能把 Turborepo 接入 CI/CD

## 核心易混点

- Turborepo 是任务编排与缓存层，不是包管理器。
- inputs、outputs、缓存命中、任务依赖要一起看。
- Turborepo filter 和 pnpm filter 的作用面不同。
- 任务能跑不代表缓存策略就设计对了。

## 必做失败演练

- 故意把 outputs 配错，观察缓存为什么命不中或命中错误。
- 故意漏掉关键环境变量输入，比较缓存脏数据如何出现。
- 故意让 filter 选择范围过宽或过窄，练习解释任务为什么没跑到或跑太多。

## 常见误区

1. 只会照抄 turbo.json，不理解任务依赖关系
2. outputs 配置错误，导致缓存命中异常
3. 把 Turborepo 当包管理器使用
4. CI 里没有利用 filter 和缓存，收益很低
