# LangSmith 学习计划

## 这个领域解决什么问题

LangSmith 用于观察、调试、评估和对比 LLM / Agent 调用链，帮助定位 Prompt、工具、检索、路由和状态流中的问题。

## 学习目标

- 理解 tracing 在 AI 系统中的价值
- 能看懂 run、span、trace 之间的关系
- 能把 LangChain / LangGraph 的调用接入 LangSmith
- 能用观测结果排查链路问题
- 能区分调试、观测、评测这三类工作

## 四阶段路径

### 阶段 1：入门理解
- Ch16-Ch17
- 重点：为什么要做 tracing，LangSmith 的定位
- 产出：能解释 trace 对调试的价值

### 阶段 2：核心能力
- Ch43
- 重点：把 LangChain 调用接入 LangSmith
- 产出：能看到链路、输入输出和中间步骤

### 阶段 3：实战落地
- 结合 LangGraph 与多工具链路
- 重点：观察多步调用、工具选择、失败点
- 产出：能用 trace 解释一次复杂调用为什么失败

### 阶段 4：巩固与排障
- 回看 Ch18-Ch20、Ch132
- 重点：指标、灰度、对比实验、线上排障
- 产出：能把 tracing 和上线后的问题定位连起来

## 核心易混点

- trace 不是普通日志替代品，重点在链路解释力。
- tracing、调试、评测、对比实验是不同工作。
- run、step、tool call 的层级关系要看清。
- 看到 trace 不等于已经定位到根因。

## 各阶段过关标准

- 阶段 1：能解释它解决的问题，并写出或画出一个最小结构。
- 阶段 2：能独立完成核心能力的小例子，并说清执行流。
- 阶段 3：能把它接到当前 AI 伴侣项目的上下游，并解释边界。
- 阶段 4：能围绕典型错误定位根因，给出最小修复和复盘。

## 必做失败演练

- 故意只接顶层 trace，不记录关键中间步骤，观察为什么看不出根因。
- 故意比较不具可比性的两次运行，练习怎样补实验条件。
- 故意让工具调用失败但 trace 标签不够，补出真正可排查的观察点。

## 改造练习

- 给现有 Agent 调用补一套最少但够用的 trace 观察点，再用它解释一次失败链路。

## 系统连接

- 上游常接 LangChain、LangGraph 和工具链路。
- 下游常接评测、Prompt 对比和线上问题排查。
- 在 AI 伴侣里常落在开发期调试链路和问题解释层。

## 常见误区

1. 把 tracing 当日志替代品，结果看不到关键业务信号
2. 只看最终输出，不看中间 run 和工具调用
3. Trace 很多，但没有围绕问题设置观察点
4. 不做对比实验，难以判断改动到底有没有帮助
