# Hono.js 学习计划

## 这个领域解决什么问题

Hono.js 负责边缘友好的 Web API、路由、中间件、Context、RPC、SSE 与服务端组合逻辑，是当前项目服务接口层的核心框架。

## 学习目标

- 理解 Hono 的请求生命周期
- 能写最小路由、中间件、错误处理和校验
- 能说清 Context、Bindings、RPC、SSE 的用途
- 能把 Hono 接到 Cloudflare Workers 运行时
- 能写出 AI API 网关或最小业务 API

## 核心易混点

- Hono 的请求生命周期、中间件顺序、Context 读写是核心主线。
- RPC、REST、SSE 是不同接口形态，不该混成一套心智。
- 边缘运行时绑定和普通 Node 进程环境不同。
- 接口跑通不等于边界划分合理。

## 各阶段过关标准

- 阶段 1：能解释它解决的问题，并写出或画出一个最小结构。
- 阶段 2：能独立完成核心能力的小例子，并说清执行流。
- 阶段 3：能把它接到当前 AI 伴侣项目的上下游，并解释边界。
- 阶段 4：能围绕典型错误定位根因，给出最小修复和复盘。

## 必做失败演练

- 故意把鉴权和校验中间件顺序写反，观察为什么行为异常。
- 故意遗漏 Bindings 或环境配置，练习定位问题是代码还是运行时。
- 故意让 SSE 或流式接口缺关键头或写错事件节奏，观察前端为什么消费失败。

## 系统连接

- 上游常接 Zod、认证、AI SDK 服务端逻辑。
- 下游常接 Cloudflare Workers、Drizzle、外部服务。
- 在 AI 伴侣里常落在服务接口层和边缘 API 层。

## 常见误区

1. 把 Hono 当成普通 Node 服务器，不考虑边缘运行时限制
2. 中间件顺序设计混乱，导致认证和错误处理失效
3. 只会写 REST，不会利用 RPC 类型推导
4. 流式响应能跑，但不知道事件流是怎么推到前端的
